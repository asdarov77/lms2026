### 🔍 Где посмотреть структуру проекта?

Да, **можно получить полную древовидную структуру папок одной командой** — прямо из терминала.

#### ✅ Для фронтенда (Vuetify + Vue 3):
Если ваш фронтенд находится в `resources/js` (стандарт для Laravel с Inertia/Vite) или в отдельной папке вроде `frontend/`, выполните в терминале:

```bash
# Если фронтенд в resources/js (часто так в Laravel+Inertia)
find resources/js -type f \( -name "*.vue" -o -name "*.js" -o -name "*.ts" \) | sort

# ИЛИ — чтобы увидеть всё дерево (включая папки)
tree resources/js
```

> 💡 Если у вас **отдельный Vue-проект** (не внутри Laravel), замените `resources/js` на путь к нему, например: `tree src`.

Если команды `tree` нет — установите её:
- **macOS**: `brew install tree`
- **Ubuntu/Debian**: `sudo apt install tree`
- **Windows (WSL)**: то же, что и Ubuntu

#### 📁 Альтернатива без терминала:
Просто откройте проводник/файловый менеджер и **вручную опишите**:
- Какие папки есть внутри `resources/js` (или `src`)?
- Внутри `views/` — какие файлы?
- Внутри `components/` — какие основные компоненты?

> Не нужно копировать код — только **названия файлов и папок**.

### 🔍 Что я уже понял из `App.vue` и `routes.js`:

#### 🧱 Архитектура фронтенда:
- **Фреймворк**: Vue 3 + Vuetify 3
- **Состояние**: Vuex (модули: `Auth`, `Ui`)
- **Локализация**: vue-i18n
- **Маршрутизация**: Vue Router с динамическими параметрами и защитой
- **Структура страниц**: компоненты лежат в `/Pages/...` с подпапками (`User/`, `Group/`, `Course/`, `Gift/`, `Settings/` и др.)

#### 🔐 Авторизация и права:
- Защита маршрутов через `hasPermission(['manage-users'], '...')`
- Условный рендеринг бокового меню в зависимости от роли (`user.role !== 'Обучаемый'`)

#### 📁 Основные модули приложения:
1. **Пользователи** (`/user/list`, `/user/edit/:id`)
2. **Группы** (`/groups/list`, `/groups/add`)

4. **Категории курсов** (`/categories`, `/categories/:idEdit`)

## 1. Обзор системы

**Название**: Авиационная LMS «Dinamika»  
**Тип**: Веб-приложение для дистанционного обучения персонала в авиационной сфере.  
**Ключевые функции**:  
- Управление курсами, категориями, учебными материалами (статический контент)  
- Групповое назначение обучения  
- Тестирование с импортом вопросов в формате GIFT  
- Ролевая модель: Обучаемый, Инструктор, Администратор  
- Календарь, избранное, настройки оценок  

**Технологический стек**:  
- **Frontend**: Vue 3, Vuetify 3, Vuex, vue-i18n  
- **Backend**: Laravel 10, PostgreSQL, Sanctum  
- **Хостинг статики**: Локальное хранение (`/private/{aircraft}/{auk}/...`)

---

## 2. Текущая архитектура

### 2.1. Структура фронтенда (`resources/js/`)
```
Pages/
├── Home.vue
├── Login.vue, Register.vue
├── User/ (UserList.vue, UserItemEdit.vue, UserChpass.vue)
├── Group/ (GroupList.vue, CreateGroup.vue, GroupItemEdit.vue, GroupLearning.vue)
├── Course/ (RegisterCourse.vue, UpdateCourse.vue, AddClass.vue)
├── Category/ (CategoryList.vue, RegisterCategory.vue, UpdateCategory.vue)
├── Gift/ (GiftImportForm.vue, ExamineItem.vue, QuestionEdit.vue, QuestionNew.vue)
├── Settings/ (GradeSettings.vue, AllSettings.vue)
├── Navigation/ (LeftSideMenu.vue, AccountMenu.vue)
└── Filemanager/, Calendar/, Errors/
```

### 2.2. Vuex-модули
- **`Auth`**: Аутентификация, права (`hasPermission`)  
- **`User`**: Пользователи, группы, смена пароля  
- **`Course`**: Курсы, категории, aircrafts, group2learnings  
- **`Ui`**: Состояние меню, язык интерфейса  

### 2.3. Бэкенд: ключевые модели и связи
```php
Aircraft → hasMany → Category → belongsToMany → Course → hasMany → Aukstructure → hasMany → Question → hasMany → Answer
User → belongsTo → Group → hasMany → Group2learning → belongsTo → Course
```

### 2.4. Проблемные зоны
1. **Дублирование логики**:  
   - Старые контроллеры (`CoursesListController`) и новые (`CourseController`)  
   - Два способа работы с группами (`AuthController@group2learning` и `Group2learningController`)

2. **Сложные маршруты статики**:  
   - Цепочка до 8 уровней вложенности: `/private/{aircraft}/{auk}/{html}/{html2}/...`

3. **Кастомная система прав**:  
   - Поле `role` в `users` + таблицы `roles`, `permissions`, `role_user`, `permissions_users`

4. **Устаревший стейт-менеджмент**:  
   - Vuex вместо современного Pinia

## 5. Примеры кода

### 5.1. Pinia-стор (courseStore.js)
```javascript
import { defineStore } from 'pinia';
import { api } from '@/services/api';

export const useCourseStore = defineStore('course', {
  state: () => ({
    courses: [],
    categories: [],
    aircrafts: []
  }),
  actions: {
    async fetchCourses() {
      const response = await api.get('/api/course');
      this.courses = response.data;
    }
  }
});
```

### 5.2. Универсальный контроллер статики (ContentController.php)
```php
public function serve(string $path)
{
    $fullPath = storage_path("app/courses/{$path}");
    if (!File::exists($fullPath)) {
        abort(404);
    }
    return response()->file($fullPath);
}
```

### 5.3. API-ресурс (CourseResource.php)
```php
public function toArray($request)
{
    return [
        'id' => $this->id,
        'title' => $this->title,
        'aircraft' => AircraftResource::make($this->whenLoaded('aircraft')),
        'categories' => CategoryResource::collection($this->whenLoaded('categories')),
    ];
}
```
